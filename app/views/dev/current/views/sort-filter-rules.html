Sorting and filtering rules

In general

The sorting, filters and pagination choices are saved in memory. When a user navigates back to a list the same rules shall be applied.

If the user has taken an action whilst outside of the list that decrements the number of results, leading to the selected page no longer existing, then the user should be redirected to the highest remaining page in the pagination set.

For example;

  1. I go to my queue and filter the list to only show schools with query number 2500, producing 251 results
  2. I navigate to the page 5 of 5 and select the only school on that page, school number 251 in those results
  3. I process the return and send the responses
  4. I return to the filters list, now showing 250 results, I am now looking at results page 4 of 4

When a user sorts a table

The sorting function is ran on the server, the page is re-rendered with the new sort criteria. The pagination is reset and reverts to the first page

When a user applies a filter

The page is re-rendered with the filtered data, the pagination is reset and reverts to the first page.
